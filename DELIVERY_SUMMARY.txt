KP SYSTEM MODULE (kp_system.py) - DETAILED FEATURES
================================================================================

CORE CLASSES (5):

1. KPSignificatorTable
   - Calculates significators for all planets and houses
   - Implements 3 KP rules:
     * Rule 1: Planet signifies houses owned by its star lord
     * Rule 2: Planet signifies houses it owns (via its sign)
     * Rule 3: Planet signifies its house of occupation
   - Methods: 7 (calculate significators, get by house, strength ranking)
   - Output: Complete planet-to-house significance matrix

2. RulingPlanets
   - Calculates the 5 ruling planets for any moment in time
   - Ruling planets:
     1. Ascendant Sign Lord
     2. Ascendant Star Lord (nakshatra)
     3. Moon Sign Lord
     4. Moon Star Lord (nakshatra)
     5. Day Lord (Sunday-Saturday mapping)
   - Applications: Event timing, prashna, verification
   - Methods: 5

3. CuspalAnalysis
   - Analyzes all 12 house cusps for event materialization
   - Determines: Sign Lord, Star Lord, Sub Lord of each cusp
   - Core principle: Event manifests if sub-lord signifies required houses
   - Methods: 7

4. KPEventPredictor
   - Predicts timing of 7 life event categories:
     * Marriage/Partnership (houses 2, 7, 11)
     * Career/Profession (houses 2, 6, 10)
     * Health Recovery (houses 1, 5, 11)
     * Health Issues (houses 6, 8, 12)
     * Birth of Children (houses 2, 5, 11)
     * Travel Abroad (houses 3, 9, 12)
     * Property Acquisition (houses 4, 11, 12)
   - Timing: Dasha/Bhukti of significators that are Ruling Planets
   - Methods: 2 (predict single, predict multiple)

5. KPAnalysis
   - Main orchestrator for complete KP analysis
   - Integrates all components
   - Output: Comprehensive multi-part report
   - Methods: 6 (including generate_report)

SUPPORTING COMPONENTS:

Enums:
  - PlanetEnum: 9 planets with Vimshottari dasha years (total 120 years)
  - SignEnum: 12 zodiac signs with sidereal lords
  - NakshatraEnum: 27 nakshatras with proper lords (Vimshottari sequence)

Data Structures:
  - PlanetPosition: Planet location (longitude, sign, nakshatra, KP sub)
  - CuspPosition: House cusp (longitude, sign, nakshatra, KP sub)
  - Significators: Result container for significator analysis

Test Function:
  - create_test_chart(): Uses Hemant's chart data
    * Ascendant: Gemini (sidereal)
    * Moon: Cancer, Ashlesha nakshatra
    * Placidus house system

KCIL SYSTEM MODULE (kcil_system.py) - DETAILED FEATURES
================================================================================

CORE CLASSES (5):

1. KPSubTable
   - Manages 249-entry KP sub division table
   - Calculation: sub_duration = zodiac_span × (planet_years / 120)
   - Uses Vimshottari proportions for full zodiac
   - Methods: 3 (get sub for degree, by index)

2. SubSubLordTable
   - Generates 2241 sub-sub lord divisions
   - Structure: 249 subs × 9 sub-subs each
   - Nadiamsha-level precision for event timing
   - Methods: 5 (includes generation and lookup)

3. KCILCuspalAnalysis
   - KCIL cuspal interlink analysis with favorability rules
   - Favorability classification:
     * Favorable (1, 3, 5, 9, 11): Event will happen
     * Neutral (2, 6, 10): Event may happen
     * Unfavorable (4, 7, 8, 12): Event blocked
   - Favorability score: 0.0-1.0
   - Methods: 6

4. KalamsnaDivision
   - Kalamsa (time division) at sub-sub level
   - Applications:
     * Precise event timing (hours/minutes)
     * Muhurta (auspicious timing) calculations
     * Planetary activation timing
   - Methods: 4

5. KCILAnalysis
   - Main KCIL orchestrator
   - Integrates all components
   - Output: Comprehensive KCIL report
   - Methods: 4

SUPPORTING COMPONENTS:

Data Structures:
  - SubDivisionInfo: KP sub or sub-sub division details
  - SubSubPosition: Position within Nadiamsha level
  - VimshottariDashaYears: Dasha duration enum

Test Function:
  - test_kcil_system(): Full KCIL analysis with event assessment
